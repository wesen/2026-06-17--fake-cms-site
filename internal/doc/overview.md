`fake-cms` is a **fake internal Content Management System API** that stands in for a real, legacy, GraphQL content API the workshop does not have access to. The real system belongs to a large French media group; its public B2B site is `https://www.20minutes-media.com/`. Because the private API is unavailable, `fake-cms` reverse-engineers the shape of the content model from the public site and replays a **deterministic, synthetic** dataset through the same kinds of queries.

This page explains what the system is made of and how the pieces fit together. Read it first; then read `help api-reference` for the field-by-field contract.

## What problem it solves

Workshop participants must build a **static site generator (SSG) plugin** that:

1. Connects to a GraphQL content endpoint.
2. Walks the content graph (articles, authors, taxonomies, media).
3. Emits static HTML pages.

A toy API with three flat fields would not teach the hard parts: pagination, nested relationships, heterogeneous content blocks, imperfect legacy data, and N+1 query patterns. So the mock deliberately mirrors a **WordPress + Yoast** CMS — the platform the real target runs — with all its legacy texture.

## The four layers

The codebase is split into layers, and dependencies point **up only**. Nothing in the GraphQL layer imports `database/sql`; nothing in the repository imports GraphQL.

```text
cmd/fake-cms/main.go         glazed Cobra CLI: serve, seed, db, version, help
internal/server              net/http: /graphql, /playground, /healthz
internal/graphql             schema-first GraphQL resolvers + global ids
internal/repo                database/sql → domain structs (batched loaders)
internal/domain              plain structs (no framework tags)
internal/migrations          embedded SQL
testdata/cms.db              committed deterministic seed
schema.graphql               THE contract (single source of truth)
doc/                         these help entries (embedded)
```

This separation is what keeps the mock testable and teachable: each layer has a focused unit-test surface, and the GraphQL contract can be reviewed without reading Go.

## Data is deterministic

The seed is generated by a **seeded PRNG** (`rand.NewSource(42)`) and contains no calls to `time.Now()`. Re-seeding an empty database produces the same rows every time, so workshop exercises have stable expected outputs. The committed artifact is `testdata/cms.db`; its content hash is stored alongside it.

## Read-only and unauthenticated

The API exposes **queries only**. There are no mutations, no subscriptions, and no authentication. This is intentional: the SSG only reads, and workshop time is finite. The trade-off — recorded as a decision — is that the mock must never be deployed publicly.

## Troubleshooting

| Problem | Cause | Solution |
|---------|-------|----------|
| `make build` fails on `ttmp/` files. | The ticket's standalone debug mains are under `ttmp/`. | Use `make build-all` (excludes `ttmp/`); see the Makefile. |
| Seeded DB looks empty. | Wrong `--path`. | `./fake-cms db query --path testdata/cms.db --query "SELECT count(*) FROM content_node"`. |
